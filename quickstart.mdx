Get your local development environment set up and start building with Vicenta, a modern Next.js 14 application with TypeScript and Tailwind CSS.

## Prerequisites

Before you begin, make sure you have the following installed:

- **Node.js** 18.17 or later
- **npm**, **yarn**, **pnpm**, or **bun** package manager
- **Git** for version control

<Note>
Vicenta uses Next.js 14.2.6 with the App Router, TypeScript 5, and Tailwind CSS 3.4.1.
</Note>

## Clone the Repository

<Steps>
  <Step title="Clone the project">
    Clone the Vicenta repository to your local machine:

    ```bash
    git clone https://github.com/Brunodistaulo/Vicenta.git
    cd Vicenta/front
    ```
  </Step>

  <Step title="Install dependencies">
    Install the required dependencies using your preferred package manager:

    <CodeGroup>
      ```bash npm
      npm install
      ```

      ```bash yarn
      yarn install
      ```

      ```bash pnpm
      pnpm install
      ```

      ```bash bun
      bun install
      ```
    </CodeGroup>

    <Tip>
    We recommend using **pnpm** or **bun** for faster installation times.
    </Tip>
  </Step>

  <Step title="Start the development server">
    Run the development server with hot reload enabled:

    <CodeGroup>
      ```bash npm
      npm run dev
      ```

      ```bash yarn
      yarn dev
      ```

      ```bash pnpm
      pnpm dev
      ```

      ```bash bun
      bun dev
      ```
    </CodeGroup>

    The development server will start on `http://localhost:3000`.
  </Step>

  <Step title="Open in your browser">
    Navigate to [http://localhost:3000](http://localhost:3000) in your browser to see your application running.

    You should see the Vicenta welcome page with links to documentation and resources.
  </Step>
</Steps>

## Project Structure

Vicenta follows the Next.js 14 App Router structure:

```
front/
├── src/
│   └── app/
│       ├── layout.tsx      # Root layout with Inter font
│       ├── page.tsx        # Home page component
│       └── globals.css     # Global styles
├── public/                 # Static assets
├── tailwind.config.ts      # Tailwind CSS configuration
├── tsconfig.json          # TypeScript configuration
└── package.json           # Project dependencies
```

<Note>
The `src/app/` directory uses the App Router pattern introduced in Next.js 13+. All routes are defined using folder-based routing.
</Note>

<Steps>
  <Step title="Open the home page">
    Open `src/app/page.tsx` in your code editor.

  <Step title="Edit the welcome message">
    Find the text that says "Get started by editing" and modify it:


## Key Technologies

Vicenta is built with modern web technologies:

- **Next.js 14.2.6** - React framework with App Router
- **React 18** - UI library with server components
- **TypeScript 5** - Type-safe JavaScript
- **Tailwind CSS 3.4.1** - Utility-first CSS framework

## TypeScript Configuration

The project uses strict TypeScript settings with path aliases:

```json tsconfig.json
{
  "compilerOptions": {
    "strict": true,
    "paths": {
      "@/*": ["./src/*"]
    }
  }
}
```

This allows you to import modules using the `@/` alias:

```tsx
import MyComponent from '@/components/MyComponent'
```
